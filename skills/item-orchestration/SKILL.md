# item-orchestration — v2 多 Item 编排

`item-orchestration` 只在存在多个被追踪的 Item 时才有意义。

它不是所有任务的默认能力，也不应反过来迫使普通任务立项。

## 适用场景

- 多个已追踪任务存在依赖
- 需要查看并行/串行关系
- 需要发现文件级或模块级冲突

## 前置条件

- `.feflow/items/` 存在
- 至少有一个活跃 Item

若没有工作区或没有活跃 Item，直接返回“当前无可编排追踪项”即可。

## 作用范围

它只负责：

- 读 Item 依赖关系
- 给出冲突和顺序建议
- 展示依赖图

它不负责：

- 强制创建 Item
- 自动改变任务状态
- 干扰未追踪任务

## 核心能力

1. 读取 `depends_on` / `blocks`
2. 检测循环依赖
3. 识别潜在并行任务
4. 提示同文件或同模块修改冲突

## 用户期望

对用户输出应聚焦：

- 哪些 Item 可以并行
- 哪些必须等待
- 哪些存在冲突风险

不要把依赖图输出成复杂治理噪音。
