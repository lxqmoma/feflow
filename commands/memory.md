# /memory

查看项目记忆概览或手动添加新的记忆条目。

## 触发方式

```
/memory              # 查看记忆概览
/memory add          # 手动添加记忆
/memory decay        # 执行记忆衰减检查与归档
```

## 执行逻辑

### 查看概览（/memory）

无参数时：

1. 读取 `.feflow/memory/` 目录下的所有记忆文件
2. 按类别分组输出概览：
   - 项目配置与约定
   - 历史决策记录
   - 团队偏好与风格
   - 已知问题与技术债
   - 复盘沉淀
3. 每条记忆展示：标题、类别、创建时间、摘要

### 手动添加（/memory add）

参数为 `add` 时：

1. 引导用户提供以下信息：
   - **类别** — 约定/决策/偏好/问题/复盘
   - **标题** — 简要描述
   - **内容** — 详细说明
   - **来源** — 来自哪个 Item 或用户手动添加
2. 生成记忆文件写入 `.feflow/memory/`
3. 确认添加成功

## 前置条件

- 项目已完成 feflow 初始化（`.feflow/` 目录存在）
- 若未初始化，提示用户先运行 `/init`

## 说明

记忆在工作流中也会自动沉淀（如复盘产出、重要决策记录）。
`/memory add` 用于手动补充自动流程未覆盖的项目知识。

### 衰减检查（/memory decay）

调用 `feflow:memory-decay` skill，检查过期记忆并按策略衰减/归档。
