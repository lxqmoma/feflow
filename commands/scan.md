# /scan

扫描当前仓库，识别技术栈、目录结构和项目关键信息。

## 触发方式

```
/scan
```

## 执行逻辑

1. 在当前会话直接按 `repo-scan` 规则执行仓库扫描
2. repo-scan 会完成以下工作：
   - 识别技术栈（框架、语言、构建工具等）
   - 分析目录结构和模块划分
   - 识别关键配置文件
   - 检测已有的测试、lint、CI 配置
   - 识别高风险区域和可能的历史包袱
3. 若项目已初始化，可将扫描结果沉淀到 `.feflow/memory/`
4. 若项目未初始化，直接以内联摘要形式输出，不把初始化作为前置阻塞
5. 输出扫描摘要，并给出适合进入 `Assist`、`Delivery` 或 `Incident` 的建议

## 命令语义

`/scan` 是聊天命令，不是 shell 命令。

不要把它描述成“缺少某个 skill 工具入口就无法执行”。

## 前置条件

- 当前目录为 Git 仓库或有效项目目录
- `.feflow/` 初始化是增强项，不是必须项

## 使用场景

- 首次接触一个项目，需要快速了解项目全貌
- 开始新的交付前，刷新对项目状态的了解
- 项目结构发生较大变化后，重新扫描
- 进入深度分析前，建立事实级仓库画像

## 首轮回复契约

首轮回复应说明会先看哪些事实来源，并直接进入扫描。

推荐形态：

- “我先看 package.json、关键配置和源码主目录，先给你一个事实级仓库画像。”

不要把 `/scan` 的开场写成初始化要求或阶段说明。

暂停预算：

- 理想值：`0`
- 最大可接受：`1`
