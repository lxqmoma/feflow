# /scan

扫描当前仓库，识别技术栈、目录结构和项目关键信息。

## 触发方式

```
/scan
```

## 执行逻辑

1. 调用 `feflow:repo-scan` skill 执行仓库扫描
2. repo-scan 会完成以下工作：
   - 识别技术栈（框架、语言、构建工具等）
   - 分析目录结构和模块划分
   - 识别关键配置文件
   - 检测已有的测试、lint、CI 配置
3. 扫描结果存储到 `.feflow/memory/` 供后续流程使用
4. 输出扫描摘要

## 前置条件

- 当前目录为 Git 仓库或有效项目目录

## 使用场景

- 首次接触一个项目，需要快速了解项目全貌
- 开始新的 Item 前，刷新对项目状态的了解
- 项目结构发生较大变化后，重新扫描
